# 论文工作流：从“看过”变成“能解释、能验证”

## 为什么不要从第一页逐字读

论文像一张城市地图。第一次先确认它是不是你要去的城市；第二次找主干道；第三次才走进每条小巷。直接逐字读，往往会在符号里迷路，却不知道论文真正解决了什么。

## 第一遍：10 分钟筛选

只看标题、摘要、引言、章节标题、图表、结论和参考文献，回答五个问题：

1. **类别**：新模型、算法、系统、测量、综述还是分析？
2. **上下文**：它建立在哪些工作之上？
3. **正确性**：关键假设看起来合理吗？
4. **贡献**：若只能保留一句话，贡献是什么？
5. **清晰度**：目前卡住是因为缺前置知识，还是论文没有讲清？

若与当前课程无关，停在第一遍完全正常。

## 第二遍：45–90 分钟理解

- 先看所有图表：坐标、单位、误差条、比较对象是否公平。
- 写出输入、输出和核心张量形状。
- 把核心公式翻译成一句普通话。
- 记录数据集、参数量、训练 token、硬件和基线。
- 找消融实验：性能究竟来自哪一部分？
- 将陌生引用放入候选池，不立即跳走。

## 第三遍：2–4 小时重建

假装论文正文消失，只根据问题重新设计方法：

- 推导关键公式或复杂度。
- 写不依赖框架细节的伪代码。
- 做最小反例：方法在哪种输入上会失败？
- 实现缩尺实验，复现“趋势”而不是追求论文的绝对数字。

## 笔记中必须分开的四栏

| 栏目 | 含义 | 示例 |
|---|---|---|
| 作者声称 | 论文明确写出的结论 | GQA 可减少 KV Cache |
| 实验证据 | 表格、图或消融真正支持的内容 | 在指定模型与硬件上吞吐提高 |
| 我的推断 | 合理但未被该实验直接验证 | 对更小模型可能也有效 |
| 未复现 | 目前无法独立确认的部分 | 大规模训练稳定性 |

使用 `papers/notes/TEMPLATE.md` 建立笔记。阅读完成日之后，在第 1、7、30 天分别进行口述、公式重写和代码回忆。

## 选论文的证据等级

从高到低记录为：正式发表且有代码/数据、正式发表、预印本且有充分消融、技术报告、只有模型卡或宣传。热度和榜单不能替代证据。

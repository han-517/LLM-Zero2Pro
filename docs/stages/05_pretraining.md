# 阶段五：预训练——模型之外，数据和测量同样重要

## 数据管道

预训练不是“下载一个大文本文件”。每条数据应经过来源记录、许可检查、语言/质量过滤、去重、污染检查、分词和混合。每一步都记录输入数量、输出数量和拒绝原因，避免静默丢数据。

精确去重找完全相同文本；近似去重找轻微改写或网页模板。重复会让模型浪费 token，也可能让评测看起来虚高。

## 资源账本

至少分别估算：

- 参数：数量 × 每参数字节。
- 梯度：通常与参数同量级。
- 优化器状态：Adam 常有一阶、二阶矩。
- 激活：与 batch、sequence、层数和隐藏维相关。
- KV Cache：主要影响自回归推理。

FLOPs 描述运算量，显存容量描述能否装下，带宽描述搬数据速度；三者不能互相替代。

## Scaling Laws

幂律表示在某个范围内，增大模型、数据或计算时损失呈平滑趋势。缩尺实验先训练多种小配置，再拟合：

```text
loss = irreducible_loss + a * size^(-alpha)
```

必须报告拟合范围、残差和误差，不把小模型曲线无条件外推到数十亿参数。

## 评测

训练 loss 回答“是否更会预测训练分布”；任务评测回答某类能力；吞吐和显存回答系统成本。一个数字不能替代另外两个。

良好消融一次只改变一个因素，并固定：数据、tokenizer、训练 token、参数/活跃 FLOPs、优化器和评测脚本。

## 故障诊断顺序

1. 用一个 batch 过拟合，检查代码链路。
2. 检查 target shift、mask、padding 和数据范围。
3. 检查 NaN/Inf、梯度和激活尺度。
4. 检查学习率与初始化。
5. 最后才扩大模型或更换优化器。

## 常见误区

- 更多 token 不等于更高质量数据。
- 不同 tokenizer 的 token 数不是同一单位。
- 验证集下降不代表所有下游任务都提升。
- 混合精度减少内存，但归约和更新仍可能需要更高精度。

# 学习方法：同一概念的六层理解

每篇讲义和每个 Notebook 都按下面顺序学习。

## 1. 生活类比与图

先回答“它想解决什么痛点”。例如注意力可以想成开卷考试：每生成一个词，都能翻阅之前的笔记；KV Cache 是把已经整理过的索引保存下来，避免每次重抄。

类比只用于建立方向，不能代替公式。

## 2. 张量形状

对每行代码标出形状，例如：

```text
x: [batch, time, d_model]
q: [batch, heads, time, d_head]
scores: [batch, heads, time, time]
```

大部分实现错误首先是形状错误。

## 3. 必要公式

只保留能指导实现和分析的公式。每个符号首次出现时都用普通话解释，并同时写清时间、显存复杂度。

## 4. 最小实现

先写不超过一屏的清晰版本，不急着融合算子或减少内存。最小实现是后续优化的“正确答案”。

## 5. 对照实验

一次只改变一个因素，并同时记录正确性、损失、时间和内存。不要用不同数据或不同训练预算比较两个架构。

## 6. 常见误区

每个主题最后维护一张错题卡，包括：错误直觉、触发样例、正确解释和能抓住该错误的测试。

## 每周完成定义

- 能不用术语向别人讲清本周核心问题。
- 能手写关键张量形状和公式。
- 能独立补完核心实现中的 TODO。
- 能解释至少一个实验为何支持或不支持结论。
- 完成一篇论文的一遍阅读；重点周完成二遍或三遍。

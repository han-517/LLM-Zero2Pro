# 逐周讲义写作与验收契约

本目录保存 48 篇真正供学习者阅读的逐周讲义。不要按文件名随机挑选：
先回到[讲义总入口](../README.md)，选择[15 周核心路线](../core_learning_path.md)
或[48 周完整路线](../full_learning_path.md)，再从路线表当前行进入本周正文。

阶段综述仍位于 `docs/stages/`，只负责说明阶段目标和导航；
`course/stages/*.yaml` 把每周唯一映射到这里的一篇正文。

## 文件命名

使用 `NN_topic_slug.md`，其中 `NN` 是两位原课程周编号。每周只能有一个
`assets.lecture`，且 48 周不得复用同一个 lecture 文件。

## 最低质量门槛

- 正文去除空白后至少 2200 个字符；这是防止提纲冒充讲义的下限，不是目标字数。
- 至少一个闭合的 fenced code block，以及三个经过核验的 https 一手来源。
- 公式、代码和结论必须明确教学 baseline 与论文完整算法或生产系统的边界。
- 本地链接必须从当前讲义文件出发能够解析，不依赖 GitHub 页面猜测路径。
- 每周要能独立回答“为什么、形状是什么、怎么算、怎样失败、如何验收”。

## 统一模板

### H1：第 NN 周——主题

课程定位：说明它承接什么、为后续什么主题提供前置。

### 二级标题 1：学习目标

写成完成后可观察、可核查的能力。

### 二级标题 2：前置知识

给出必要复习和不需要提前掌握的内容。

### 二级标题 3：核心直觉

使用一个贯穿本周的直觉模型，但不能用比喻代替机制。

### 二级标题 4：张量与数据契约

写明 shape、dtype、mask、单位、输入输出和非法边界。

### 二级标题 5：公式推导或算法机制

逐步解释符号、假设和推导，不只粘贴最终公式。

### 二级标题 6：手算或数值示例

至少给出一个能人工核对的微型例子。

### 二级标题 7：最小代码实现

代码应能离线运行，并指出它省略的生产能力。

### 二级标题 8：反例、常见误区与调试

主动构造错误输入，给出症状、原因和定位顺序。

### 二级标题 9：主流工作与实现边界

区分基础共识、公开模型采用和前沿预印本。

### 二级标题 10：实验与 Notebook 对照

链接对应 Notebook、互动图和 starter，说明先预测什么。

### 二级标题 11：验收标准

给出断言、数值容差、rubric 和本周交付物。

### 二级标题：一手来源

优先原论文、官方技术报告、模型卡、代码仓库和框架文档。

# 毕业项目：在公平预算下比较三种语言模型

## 研究问题

在相同数据、训练 token 数和近似活跃 FLOPs 下，Dense Transformer、注意力优化模型和 MoE 的损失、吞吐、内存与失败模式有何不同？

## 默认缩尺配置

- 数据：TinyStories 的固定小切片，或仓库内置字符级样例。
- CPU：每个模型 0.5–5M 参数，先过拟合一个 batch，再进行短训练。
- 免费 GPU：可扩到 20–50M 参数，但不得改变比较规则。
- 变体：Dense；GQA/线性注意力二选一；4–8 专家 Top-2 MoE。

## 公平比较规则

- 使用相同 tokenizer、训练/验证切分、token 数和评测脚本。
- 同时报告总参数与每 token 活跃参数，不只报告模型文件大小。
- 固定序列长度、batch、dtype、预热次数和随机种子。
- 至少运行三个种子；资源不足时允许一个种子，但必须标注限制。
- 不把不同硬件的绝对速度放进同一排名。

## 最终报告结构

1. 问题与假设。
2. 模型和预算匹配方法。
3. 正确性测试与实验设置。
4. 训练/验证损失、吞吐、峰值内存和路由分布。
5. 生成样例与失败案例。
6. 哪些结论有证据，哪些只是推测。
7. 若有更多算力，下一步会验证什么。

项目不要求复现大模型榜单；目标是展示严谨、透明、可重复的推理过程。
